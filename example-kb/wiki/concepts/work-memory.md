# Work Memory

## Summary

Work memory is the durable research context that the user and agent have already figured out together and want to reuse later. It includes stable conclusions, project theses, reusable frameworks, known dead ends, and open questions that materially shape future work.

## Core Points

- It stores reusable knowledge, not routine chat residue.
- It should be grounded in sources, projects, insights, or explicit synthesis.
- It can pass through a candidate note first when the idea matters but is still unresolved.

## Boundary Test

Use this concept when a note would help future topic-level reasoning, comparison, or project direction.

Do not treat something as work memory when it is only:

- a tone preference
- a one-off task checkpoint
- a raw transcript fragment with no reusable synthesis

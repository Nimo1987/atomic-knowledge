# Agent Memory Boundary

## Summary

This project tracks the line between a long-term `work memory` system and broader memory products that prioritize personalization or general recall.

## Current Thesis

The system stays healthier when durable conclusions live in formal wiki pages, reusable operating behavior lives in procedure pages, and candidate notes stay as a buffer for unresolved but valuable thinking. That keeps retrieval clean and prevents the knowledge base from turning into a transcript archive.

## Why This Matters

- It protects the product from drifting into `remember everything` behavior.
- It gives the agent a predictable retrieval path through `active -> recent -> index -> project -> procedure -> insight`.
- It keeps provisional material visible without letting it outrank settled pages.

## Comparison Frame

- [Work Memory](../concepts/work-memory.md): the target layer for compiled research conclusions, reusable frameworks, and live project theses.
- [Mem0](../entities/mem0.md): the comparison target that helps explain what a broader memory layer looks like.
